Plug n’ Code

Plug n’ Code is a web application which aims to provide a pluggable and dynamic development environment with a clean and easy to use and user interface.

Why Plug n’ Code?

There are many solutions out there which enable the users to run simple or slightly advanced projects online, but all of them run short when it comes to these issues,

  • No support for user built plugins, thus rendering the project non-extensible

    In Plug n’ Code, any user after going through a minimal authentication process, can create and publish a plugin in the marketplace to add support for a specific language. This enables the users to go beyond the pre-defined capabilities of the application.

    Also, each plugin goes through a thorough screening process to ensure no harmful or ill-intended code goes into the system. It goes through automated tests to test the structure of the plugin, it is then tested by the admins to test the functionality provided by the plugin.

  • No support for complex execution environments with multiple languages

    In Plug n’ Code the goal is to provide pre-defined environments as well as the option to tailor the execution environment specific to the project’s needs.

  • No support for secure execution of code

    Each build of a project in Plug n’ Code runs on a separate isolated docker instance to make sure no system specific issues interfere with the job. This also ensures safety of the project data and all kinds of output or logs are directly forwarded to the user’s browser window without being persisted in any kind of data storage medium.

Execution environment

By default the users will be provided a set of languages to be set as an execution environment for their current project.
Complex execution environments with multiple languages and custom build scripts can be created by the user.
New languages can be published (with appropriate rights) or installed as plugins.

Plugins

Plugins will be used to add new languages to a user’s workspace
Plugins can be installed from the marketplace by any user, and details of the plugins will be saved in the user’s project directory itself.

Marketplace

Here all the published plugins will be available for installation.
Each plugin will have its developer’s name and a unique developer signature.
Plugins can be published only if they are approved by the admin, and can be deleted by the plugin’s developer or admin.
