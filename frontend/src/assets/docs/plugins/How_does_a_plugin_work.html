What is a plugin?

In Plug n' Code, a plugin consists of two parts,

  • A dockerfile
  • Run commands

How does a plugin work?

Dockerfile holds the settings to be used while creating a docker image.

This is done when a new project is created, or an existing one is either imported or saved.


Run commands are the more customizable part of a plugin.

Where a dockerfile is only used when building a docker image, run commands are used to handle the execution of the project on each run.

These can be customized from the settings menu on the index page.
